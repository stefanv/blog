{{/* Drop-in replacement for Hugo's figure shortcode as of 2020-05-02 that uses img srcset to enable browsers to download only the resolution that they need. (Not quite drop-in, because this only works for images that are part of the page bundle. It will break otherwise.) Improvements that are currently out of reach of my Hugo template programming "skills": - gracefully handle images that are not in page bundle, i.e. no image processing available - use a single configurable sizes array, and derive everything from there See https://cpbotha.net/2020/05/02/drop-in-replacement-for-hugo-figure-shortcode-with-img-srcset-support/ - original srcset img shortcode from: https://laurakalbag.com/processing-responsive-images-with-hugo/ - original hugo figure shortcode from: https://github.com/gohugoio/hugo/blob/master/tpl/tplimpl/embedded/templates/shortcodes/figure.html - mashing together and srcset logic fixes by Charl P. Botha https://cpbotha.net/ */}} {{/* get file that matches the filename as specified as src="" in shortcode */}} {{ $src := .Page.Resources.GetMatch (printf "*%s*" (.Get "src")) }} {{/* resize the src image to the given sizes; this will be cached by hugo */}} {{ $tiny := ($src.Resize "480x") }} {{ $small := ($src.Resize "800x") }} {{ $medium := ($src.Resize "1200x") }} {{ $large := ($src.Resize "1500x") }} {{- if .Get "link" -}} {{- end }} {{- if .Get "link" }}{{ end -}} {{- if or (or (.Get "title") (.Get "caption")) (.Get "attr") -}} {{- end }}
